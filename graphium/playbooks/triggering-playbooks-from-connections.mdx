## Overview

In addition to manual and scheduled triggers, playbooks can fire automatically in response to external events — without anyone clicking a button or waiting for a scheduled time. Two trigger types enable this:

- **Connection triggers** — the playbook fires when something happens in a connected external system, such as a new row being added to a Google Sheet, a case being created in your EMR, or a file arriving on an SFTP server.
- **Vault event triggers** — the playbook fires when a record is created, updated, or deleted in a specific vault table.

Both trigger types are configured in the playbook editor and become active as soon as the playbook is published.

---

## Connection Triggers

A connection trigger links your playbook to a specific event on one of your vault's external connections. When that event occurs, the connection sends a notification to Graphium and a playbook run starts automatically with the event data available to the AI.

### Step 1: Select the Connection Trigger type

In the playbook editor, click the **trigger card** in the Playbook Outline and select **Connection** as the trigger type.

### Step 2: Select a Connection and Trigger Event

Two dropdowns appear:

**Connection** — Select which vault connection the trigger should listen on. Only connections that have been enabled in the Playbook Context are available here. If the connection you need is not listed, open **Playbook Context** and toggle it on first.

**Trigger event** — Select the specific event to listen for. Available events depend on the connection type. Examples:

| Connection type | Example trigger events |
|---|---|
| Google Sheets | Row Added, Row Updated |
| Graphium EMR | Case Created, Case Updated, Case Signed |
| FTP/SFTP | New File Uploaded, File Modified |

<Frame>
  <img src="/images/graphium/playbooks/connection-trigger-event.png" alt="Selecting a connection and trigger event" />
</Frame>

---

### Step 3: Configure Trigger Parameters

Most connection triggers require you to specify which resource to watch — for example, which spreadsheet and which sheet tab within it, or which facility within your EMR. These parameters are configured using dynamic dropdowns that load available options directly from the external service.

Click each parameter dropdown to see the available options fetched from your connected account.

<Frame>
  <img src="/images/graphium/playbooks/connection-trigger-parameters.png" alt="Dynamic trigger parameter dropdowns" />
</Frame>

**Cascading parameters:** Some parameters depend on the value of a previous one. For example, selecting a Spreadsheet first loads the available Sheet tabs within that spreadsheet. Complete parameters in order from top to bottom — dependent dropdowns will load automatically once their parent value is selected.

> **Note:** Parameter options are fetched live from your connected account. If you do not see the resource you expect, confirm that your connection has the correct permissions and try refreshing the options.

---

### Step 4: Add a Description (Optional)

You can add an optional **Description** to the trigger to document what event this playbook is listening for and why.

---

### What Happens at Publish

When you publish a playbook with a connection trigger, Graphium automatically **deploys** the trigger — registering it with the external service so it begins listening for the configured event. No manual setup is required outside Graphium.

When you **disable** a connection-triggered playbook, the trigger is paused and the external service stops sending events. Re-enabling the playbook re-activates the trigger.

> **Note:** If the trigger deployment fails (for example, if the connection's credentials have expired), the playbook will show an error state. Check the connection's status in the Connections section and re-publish once it is healthy.

## Vault Event Triggers

A vault event trigger fires your playbook whenever a record change occurs in a specific vault table — without any external connection required.

### Step 1: Select the Vault Event trigger type

In the playbook editor, click the **trigger card** and select **Vault Event** as the trigger type.

---

### Step 2: Select a Table and Event Type

**Table** — Select the vault table to monitor.

**Event type** — Select which record change should fire the playbook:

| Event type | When it fires |
|---|---|
| **Record Added** | A new record is created in the selected table |
| **Record Updated** | An existing record's data is changed |
| **Record Deleted** | A record is deleted from the table |

<Frame>
  <img src="/images/graphium/playbooks/vault-event-trigger.png" alt="Vault event trigger configuration showing table and event type selection" />
</Frame>

> 💡 **Tip:** Vault event triggers are powerful for real-time validation and alerting workflows — for example, automatically checking a new case record for completeness the moment it is created, or flagging an updated record that no longer meets compliance criteria.

---

### What Happens at Publish

Vault event triggers become active immediately when the playbook is published. No external deployment is required — Graphium monitors the selected table directly. Disabling the playbook pauses the trigger; re-enabling it resumes monitoring.

---

## Run History for Automatically Triggered Runs

Runs started by connection and vault event triggers appear in the **Run History** panel alongside manual and scheduled runs. Each row shows the trigger type so you can identify how the run was started.

The event payload that triggered the run — for example, the row data from Google Sheets, the case record from the EMR, or the vault record that was created — is available to the AI during execution. You can reference it in your step instructions.

## Troubleshooting

| Issue | Likely Cause | What to Do |
|-------|--------------|------------|
| Playbook is not firing when expected | The playbook may be disabled, or the trigger may be in an error state | Check that the playbook is enabled and that the trigger card shows no error indicator |
| Connection trigger shows an error state after publishing | The connection's credentials may have expired, or the external service rejected the deployment | Open Connections, verify the connection is healthy, then re-publish the playbook |
| A vault event trigger did not fire for a specific record change | The trigger may be configured for a different table or event type | Open the trigger configuration and confirm the table and event type match the change you expected |
| Parameter dropdowns are empty or not loading | The connection may not have permission to list those resources, or the connection credentials may have expired | Check the connection status and ensure it has the correct permissions for the resource you are trying to configure |
| Duplicate runs are appearing for the same event | A known external service may be retrying its webhook delivery | Graphium deduplicates events automatically — if duplicates are appearing, contact support |
