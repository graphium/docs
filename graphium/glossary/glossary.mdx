## Overview

A quick reference for key terms used throughout Graphium AI.

---

**Action (Autonomous Action)** -- A scheduled or event-driven task that an AI assistant performs automatically, without being prompted. Configured per-assistant under the Actions tab. Examples: nightly reports, weekly summaries, event-triggered scans.

**Admin** -- The highest vault permission role. Full access to manage the vault, its users, and all data. Can configure tables, schemas, connections, workflows, assistants, and permissions.

**Assistant (AI Assistant)** -- A configurable AI agent scoped to a specific vault's data. Assistants are configured with a system prompt, table access controls, and connection actions. Users interact with them through natural language chat. Also referred to as agents.

**Auto ID** -- A field type that automatically generates a unique identifier for each record (e.g., AAC-01001, REQ-0200).

**Builder** -- A vault permission role. Can create and modify tables, schemas, forms, and workflows, but cannot manage users or vault-level settings. Also refers to Builder Mode, the UI state that exposes configuration tools.

**Channel** -- An external communication platform (Microsoft Teams, Slack, or SMS) where an AI assistant is deployed as a bot. Users can message the assistant directly from these platforms.

**Connection** -- A bidirectional data pipeline between a Graphium vault and an external system (EMR, spreadsheet, database, CRM, etc.). Connections sync data in and out and expose actions that assistants and workflows can use.

**Connection Action** -- A specific event or operation exposed by a connection type. Examples: "Case Created" (Graphium EMR), "Row Added" (Google Sheets), "Survey Completed" (Graphium Surveys). Actions can trigger workflows or be made available to AI assistants.

**Contributor** -- A vault permission role. Can add and edit data within tables but cannot modify the vault's structure (schemas, workflows, connections). Also refers to Contributor Mode, a streamlined data-entry UI.

**Data View** -- The default table tab showing records in a tabular format with columns, pagination, and a Load More option.

**Field** -- A single property within a table schema. Each field has a name, a key (snake_case identifier), and a type (Text, Number, Date, etc.).

**Group** -- A field type that acts as a container for child fields, enabling hierarchical/nested data structures. Example: a "Patient Demographics" group containing Date of Birth, Gender, and Insurance fields.

**Image** -- A field type for storing image data. Used for attaching photographs, documents, or visual records to a record.

**Managed Table** -- A table whose schema is owned by an external connection (e.g., "Managed by Graphium EMR"). The schema is read-only and cannot be modified within Graphium; changes must be made in the source system.

**Organization** -- See Workspace.

**Record** -- A single data entry within a table. Records have field values that conform to the table's schema and may include a status lifecycle (e.g., Submitted, In Review, Ready to Bill, Paid).

**Reference** -- A field type that links to a record in another table, functioning as a foreign key. Example: a "Provider" field in the Cases table that references the Providers table.

**Sample Vault** -- A pre-loaded vault with demo data, available for new users to explore tables, connections, and workflows before creating their own.

**Schema** -- The structure definition of a table, consisting of an ordered list of fields with their names, keys, and types. Schemas can be hierarchical (using Groups) and may include cross-table relationships (using References).

**Schema View** -- A table tab that displays the field structure as an editable list, showing each field's name, key, type, and hierarchy.

**Signature** -- A field type for capturing digital signatures. Used for provider sign-offs and document authentication.

**System Prompt** -- The configuration text that defines an AI assistant's role, personality, domain expertise, and behavioral guidelines. Set in the General tab of assistant configuration.

**Table** -- A structured data collection within a vault, defined by a schema of hierarchical properties. Vaults can contain multiple tables (e.g., Cases, Procedures, Providers).

**Task** -- A prioritized action item surfaced by an AI assistant, workflow, or manual creation. Tasks have a title, description, priority level (Urgent, High, Medium, Low), linked records, and assignee. They represent issues that need human attention.

**Text[]** -- A field type representing an array of text values. Example: a Procedure Codes field that can hold multiple CPT codes for a single case.

**Trigger** -- The event or schedule that initiates a workflow. Types include new record creation, connection events, and time-based schedules.

**Vault** -- The central organizing unit in Graphium. A secure workspace that contains tables, connections, AI assistants, workflows, and tasks. Each vault is scoped to a specific dataset or use case.

**Viewer** -- The lowest vault permission role. Read-only access to view data. Cannot add, edit, or configure anything. Suitable for auditors, external reviewers, or stakeholders.

**Workflow** -- An automated process that triggers based on events or schedules within a vault. Workflows can fire on new records, connection events, or time-based schedules. Found under the Automations section alongside Connections.

**Workspace** -- The top-level grouping in Graphium. Users can belong to multiple workspaces and switch between them. Each workspace has its own vaults, assistants, and settings. Also called an Organization.
