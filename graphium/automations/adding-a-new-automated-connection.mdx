## Overview

Adding a connection links your vault to an external data source such as an EMR, file server, or spreadsheet. Once connected, data can flow in and out of the vault automatically based on your workflow configuration.

## When to Use This

Use this when you need to connect your vault to a new external system for the first time.

## Prerequisites

- Admin or Builder vault role
- You must know which connection type you need (e.g., Graphium EMR, Google Sheets, FTP/SFTP)

## Step-by-Step Instructions

### Step 1: Open the Data Connections Page

Navigate to your vault → **Automations** → **Connections**.

### Step 2: Click + Add Connection

Click the **+ Add Connection** button in the top-right of the Data Connections page. The **Add Connection** picker opens.

### Step 3: Find the Connection Type

Use the **Search connections...** bar to search by name, or browse the categorized list:

| Category | Connection Types |
|----------|-----------------|
| **EMR / EHR** | Graphium EMR, Epic |
| **File Server** | SFTP File Server |
| **Spreadsheet** | Google Sheets |

Click **Details** on any connection type to learn more before adding.

### Step 4: Select and Configure

Click the connection type you want to add, then complete the configuration form — including the connection name, authentication credentials, and sync settings.

### Step 5: Save the Connection

Click **+ Add Connection** to save. The new connection appears in the Data Connections list.

## What to Expect

After adding, the connection card or row displays in the Data Connections list with a status badge. If authentication succeeds, the status shows **"connected"**.

## Common Issues & Troubleshooting

| Issue | Likely Cause | Solution |
|-------|-------------|----------|
| + Add Connection button is not visible | Insufficient vault role | Contact your vault Admin to request Builder or Admin access |
| Connection status does not show "connected" after setup | Authentication credentials may be incorrect | Click the connection to open its detail view and review the Authentication settings |
