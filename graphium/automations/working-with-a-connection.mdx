## Overview

Each connection has a detail view that shows its full configuration, sync settings, and an activity log of all events triggered by the connection. From here you can review and update settings, test connectivity, investigate activity, and delete the connection when it's no longer needed. You must have an Admin or Builder vault role to access and manage connection details.

## Opening a Connection

In the vault sidebar, click **Connections**. In card view, click anywhere on the connection card. In list view, click the **">"** chevron at the end of the connection's row. The connection detail view opens with two panels:

**Left panel — Configuration:**
- **Connection Name** — the display name for this connection
- **Code Name** — the internal identifier used in workflows
- **Authentication** — credentials and auth settings for the external system
- **Sync Settings** — auto-sync toggle and sync interval
- Connection-specific settings (vary by connection type)

**Right panel — Activity Log:** shows all events triggered by the connection (see [Using the Activity Log](#using-the-activity-log) below).

The header bar provides **Test** and **Disable** buttons, as well as a **"..."** menu for additional actions.

---

## Editing Connection Settings

Connection settings can be updated at any time from the left panel. Changes take effect immediately.

1. Open the connection detail view.
2. **Update the Connection Name** — click into the **Connection Name** field and type a new name. Changes save automatically when you click away.
3. **Update Authentication** — in the **Authentication** section, update your credentials. For OAuth-based connections (such as Google Sheets), click the re-authenticate button to go through the authorization flow again.
4. **Adjust Sync Settings** — toggle **Auto-sync** on or off, and set the **Interval** for how frequently syncs occur.
5. **Update connection-specific settings** — scroll down for any settings specific to this connection type (e.g., spreadsheet name, server path, or EMR endpoint).

Most settings save automatically when you click away from a field. After updating authentication, run a **Test** to confirm the connection is still working correctly.

---

## Testing a Connection

The **Test** button runs a real-time health check to verify the connection is active and authentication is valid. Use this after setting up a new connection, updating credentials, or when you suspect a connection has stopped working.

1. Open the connection detail view.
2. Click the **Test** button in the top-right header.
3. The result displays inline — a successful test confirms the connection is active; a failed test indicates a connectivity or authentication issue.

After clicking Test, a **log** event appears in the Activity Log confirming whether the health check passed or failed.

---

## Using the Activity Log

The Activity Log in the right panel gives you a real-time record of everything that has happened with the connection — data syncs, triggered workflows, health checks, and errors.

**Event type badges:**

| Type | Color | Description |
|------|-------|-------------|
| **trigger** | Blue | A workflow trigger was fired |
| **action** | Green | A workflow action was executed |
| **sync** | Teal | A data sync completed |
| **log** | Grey | A health check or informational event |
| **error** | Red | An error occurred (auto-recovered or requires attention) |

- **Filter by type** — click the **All Types** dropdown and select a specific event type
- **View details** — click any activity row to expand the full event details
- **Navigate pages** — use **Previous** and **Next** to page through entries (up to 15 per page)

## Deleting a Connection

Deleting permanently removes the connection from the vault and stops all associated data syncs. Any workflows that relied on the connection will stop firing. This action cannot be undone. If you only need to pause syncing temporarily, use [Disable](/graphium/automations/enabling-and-disabling-an-automated-connection) instead.

1. Open the connection detail view.
2. Click the **"..."** menu in the top-right corner.
3. Select **Delete Connection** (shown in red). A confirmation modal appears.
4. Confirm the deletion. The connection is permanently removed.

## Troubleshooting

| Issue | Likely Cause | Solution |
|-------|-------------|----------|
| Authentication section is not editable | Connection may be in a disabled state | Re-enable the connection first, then update credentials |
| Test fails immediately | Authentication credentials are invalid or expired | Update credentials in the Authentication section and test again |
| Test times out | External system is unreachable | Check that the external system is online and accessible |
| Activity log is empty | No sync activity has occurred yet | Trigger a test sync using the **Test** button |
| Error entries appearing frequently | Authentication failure or external system issue | Review the error description and check the Authentication section |
| Delete Connection is not visible in the "..." menu | Insufficient vault role | Only Admin and Builder users can delete connections |
| Workflows stopped working after deletion | They were relying on the deleted connection | Reconfigure the affected workflows with a new or existing connection |
