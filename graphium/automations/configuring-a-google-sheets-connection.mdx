## Overview

The Google Sheets connection type links a vault to a Google Sheets spreadsheet, enabling automated reading, writing, and monitoring of spreadsheet data. It is useful for teams that manage data in Google Sheets and want to sync that data into Graphium vaults without manual imports. Use this when your team uses Google Sheets to collect or track data and you want Graphium to automatically pull rows into a table, push updated records back to the sheet, or trigger workflows when the sheet changes. To configure this connection, you must have an Admin or Builder vault role with access to the vault's Connections section, a Google account with access to the target spreadsheet, and permission to authorize Graphium to access your Google Sheets.

## Step-by-Step Instructions

### Step 1: Open the Add Connection Picker

In the vault sidebar, click **Connections**, then click **+ Add Connection**.

### Step 2: Select Google Sheets

Under the **Spreadsheet** category, select **Google Sheets**. Click **Details** to learn more, or click **+ Add Connection** to proceed.

<Frame>
  <img src="/images/graphium/connections/google-sheets-connection.png" alt="Google Sheets connection" />
</Frame>

### Step 3: Accept Pipedream Integration

Graphium AI uses Pipedream to connect your Google Sheets spreadsheet. When the Pipedream Integration popup appears, simply click **Continue**.

### Step 4: Sign in with Google

Pipedream will open a Google sign-in prompt to authorize Graphium AI.

- Sign in with the Google account that has access to the target spreadsheet.
- Review the permissions Pipedream requests on Graphium's behalf and grant the requested read and write access to your sheets.
- When the Google flow finishes, you'll return to Pipedream with a confirmation that the account is connected.

### Step 5: Configure Spreadsheet Settings

Enter the specific spreadsheet details — such as the sheet name and data range — so Graphium knows which data to sync.

### Step 6: Set Sync Settings

Configure how often Graphium should sync with Google Sheets:

- **Auto-sync** — toggle on to enable automatic syncing
- **Interval** — set the sync frequency (e.g., every 5 minutes)

<Frame>
  <img src="/images/graphium/connections/auto-sync-settings.png" alt="Auto-sync settings" />
</Frame>

### Step 7: Save the Connection

If successful, the Google Sheets connection will appear in the Data Connections list with a status badge. Next, click the **Test** button in the top right corner to sync your data from your Google Sheet. You will want to ensure the Google Sheets connection shows **"Healthy"** otherwise your data will not sync or migrate over to Graphium AI.

<Frame>
  <img src="/images/graphium/connections/test-google-sheet.png" alt="Test Google Sheet connection" />
</Frame>

## What to Expect

Once connected, the status badge shows **"healthy"** and data begins syncing based on your configured interval. Graphium can now read rows from and write rows to the connected sheet. Activity such as row added and row updated triggers appears in the connection's Activity Log.

**Next steps:**

- Map vault tables to your new Google Sheets data source so rows can flow into Graphium AI.
- Set up workflows or autonomous actions that react to row changes in the sheet.
- Ask your AI Assistant questions about the synced sheet data once tables are populated.

## Troubleshooting

| Issue | Likely Cause | Solution |
|-------|-------------|----------|
| Pipedream Integration popup does not appear | Browser is blocking popups for Graphium AI | Allow popups for the Graphium AI domain and retry Step 3 |
| Google sign-in window closes immediately or shows an error | Third-party cookies are blocked or the browser is in strict privacy mode | Enable third-party cookies for `accounts.google.com` and `pipedream.com` and retry, or try the flow in a standard browser window |
| OAuth authorization fails | Google account lacks access to the sheet | Ensure the Google account used has at least viewer access to the target spreadsheet |
| Permissions prompt does not include Sheets access | Wrong Google account selected during sign-in | Click the account avatar in the Pipedream window, switch to the correct account, and re-authorize |
| Connection status does not show **"Healthy"** after clicking Test | OAuth succeeded but the configured sheet or range is invalid | Open the connection detail and verify the spreadsheet ID, sheet name, and data range, then re-run Test |
| **Google Sheets** option is missing from the **Spreadsheet** category | Connection type may not be enabled for your workspace | Contact [support@graphium.ai](mailto:support@graphium.ai) to confirm availability |
| **+ Add Connection** button is not available in the vault sidebar | Insufficient vault role | Ask a vault Admin to grant Admin or Builder access |
| Auto-sync toggle cannot be enabled | Connection is in an unhealthy state | Resolve any authentication errors with Test first, then enable Auto-sync |
| Status shows "connected" but no data syncs | Incorrect sheet name or range configured | Open the connection detail and verify the spreadsheet-specific settings |
| Auth expires unexpectedly | Google token revoked or the password changed | Re-authenticate via the Authentication section in the connection detail view |
| Row changes in the sheet are not triggering workflows | Auto-sync is disabled | Open the connection detail and enable the Auto-sync toggle |
| Sync completes but no records appear in vault tables | Tables are not yet mapped to the Google Sheets data source | Open the table settings and configure the Sheets source mapping |
| Duplicate rows appear in the vault on each sync | No unique key configured for the mapped table | Set a unique key column in the table mapping so syncs deduplicate on the source row identifier |
