## Overview

Graphium AI surfaces in-app notifications when important events occur in your workspace or vault — connection failures, playbook issues, and membership changes. Notifications appear in the bell icon in the global navigation bar and are personal to each user; what you read or dismiss does not affect anyone else's notification feed.

---

## What Triggers a Notification

Notifications are generated by four categories of events:

**Connection health changes**

When a connection's health status changes to UNHEALTHY, ERROR, or DEAD, Graphium sends a notification to alert vault Admins and Builders. This covers any automated connection type — Graphium EMR, FTP/SFTP, Google Sheets, and others. A healthy connection that degrades without warning is the most common reason to check your notification feed.

**Playbook run failures and approval requests**

When a playbook run changes to a FAILED state or reaches an AWAITING_APPROVAL step (an approval gate requiring human sign-off before continuing), a notification is sent to vault Admins and Builders. This ensures the right people are alerted when automation stops or needs intervention, without having to monitor the Run History panel manually.

**Workspace member role and status changes**

When a workspace member's role or account status changes — for example, if someone is promoted, demoted, or deactivated — the affected member receives a notification, as do workspace Owners and Admins.

**Vault member role and status changes**

When a vault member's role changes — for example, if a user is added to a vault as a Builder or has their role changed — the affected member is notified, along with vault Admins.

---

## Who Receives Each Notification

Notifications are scoped by the context of the event:

- **Connection health** and **playbook run** notifications are delivered to Admins and Builders within the vault where the connection or playbook lives.
- **Workspace membership** notifications are delivered to workspace Owners and Admins, as well as the affected member.
- **Vault membership** notifications are delivered to vault Admins and the affected member.

If you are not an Admin or Builder in a vault, you will not receive notifications for events in that vault.

---

## V1 Delivery: In-App Only

The current version of notifications is in-app only. Notifications appear in the bell dropdown in the global navigation bar within the Graphium AI web application. There is no email, Slack, or push notification delivery in V1.

Notification creation is also non-blocking — a notification being generated for an event never delays or prevents that event from completing. If a playbook run fails, the run completes its failure state first; the notification follows independently.

---

## Per-User Notification State

Every notification is tracked independently for each user. When Graphium generates a notification for an event — say, a connection failure in a vault with three Admins — each Admin receives their own separate copy of that notification with their own read and dismissed state.

Reading, marking as read, or dismissing a notification in your feed has no effect on any other user's notification feed. The same event notification can appear as unread for one user and already dismissed for another, simultaneously.
