## Overview

The **SFTP (Password Auth)** connection type links a vault to a remote SFTP server using username and password authentication. Use this when your organization stores data files — such as exports, reports, or intake files — on an SFTP server and you want Graphium AI to automatically read from or write to that server using standard password-based credentials. To configure this connection, you must have an Admin or Builder vault role with access to the vault's Connections section, valid server credentials (hostname, username, and password), and network access to the file server from Graphium AI.

> **Note:** If your SFTP server requires SSH key authentication instead of a password, use the [SFTP (Key-Based Auth)](/graphium/connections/configuring-an-sftp-key-based-auth-connection) connection type instead.

## Step-by-Step Instructions

### Step 1: Open the Add Connection Picker

In the vault sidebar, click **Connections**, then click **+ Add Connection**.

<Frame>
  <img src="/images/graphium/connections/connections-sidebar.png" alt="Connections sidebar" />
</Frame>

<Frame>
  <img src="/images/graphium/connections/add-connection.png" alt="Add connection" />
</Frame>

### Step 2: Select SFTP (Password Auth)

Under the **File Server** category, select **SFTP (Password Auth)**. Click **Details** to learn more, or click **+ Add Connection** to proceed.

<Frame>
  <img src="/images/graphium/connections/add-sftp-password-connection.png" alt="Add SFTP Password Auth connection" />
</Frame>

### Step 3: Accept Pipedream Integration

Graphium AI uses Pipedream to connect your SFTP server account. When the Pipedream Integration popup appears, simply click **Continue**.

<Frame>
  <img src="/images/graphium/connections/confirm-pipedream.png" alt="Confirm Pipedream integration" />
</Frame>

### Step 4: Configure SFTP Authentication

Enter your server credentials in the Pipedream authentication form:

- **Hostname** — The network address of the SFTP server, either a domain name (e.g., `sftp.examplehospital.com`) or an IP address (e.g., `198.51.100.42`). Your server administrator can provide this.
- **Username** — The account name the SFTP server recognizes for Graphium AI to log in as. Your server administrator can provide this.
- **Password** — The password for the username above. Your server administrator can provide this.

<Frame>
  <img src="/images/graphium/connections/configure-pipedream-sftp-password.png" alt="Configure Pipedream SFTP password authentication" />
</Frame>

### Step 5: Save the Connection

If successful, you should see a **Successful Connection** notification appear from Pipedream. Simply click **Continue**.

<Frame>
  <img src="/images/graphium/connections/sftp-pipedream-successful.png" alt="SFTP Pipedream successful connection" />
</Frame>

Next, click the **Test** button in the top right corner to verify the connection. Ensure the connection shows **"Healthy"** — otherwise data will not sync or transfer correctly.

<Frame>
  <img src="/images/graphium/connections/sftp-healthy.png" alt="SFTP connection healthy status" />
</Frame>

### Step 6: Set Sync Settings

Configure how often Graphium AI should check the server for new or updated files:

- **Auto-sync** — toggle on to enable automatic monitoring
- **Interval** — set the check frequency (e.g., every 5 minutes)

<Frame>
  <img src="/images/graphium/connections/auto-sync-settings.png" alt="Auto-sync settings" />
</Frame>

## What to Expect

Once connected, the status badge shows **"healthy"** and data begins syncing based on your configured interval. Playbooks can now use this connection to trigger on new files detected on the server or to push files from Graphium AI to the server. All file transfer activity appears in the connection's Activity Log.

**Available triggers:**
- **Watch Remote Directory** — fires when a new file appears in the configured remote directory

**Available actions:**
- Delete File
- Download File
- List Directory
- Upload File

**Next steps:**

- Map vault tables to your SFTP data source so incoming files can flow into Graphium AI.
- Set up playbooks or autonomous actions that react to new or updated files.
- Ask your AI Assistant questions about the synced file data once tables are populated.

## Troubleshooting

| Issue | Likely Cause | Solution |
|-------|-------------|----------|
| Pipedream Integration popup does not appear | Browser is blocking popups for Graphium AI | Allow popups for the Graphium AI domain and retry Step 3 |
| **"Successful Connection"** notification does not appear after entering credentials | Invalid credentials or the server is unreachable from the public internet | Confirm the hostname resolves, the server is online, and the username and password are correct |
| Status shows "error" after setup | Incorrect host, username, or password | Double-check server details in the Authentication section |
| Connection times out during Test | Firewall or network restriction | Confirm the server allows inbound connections from Graphium AI's IP range |
| Connection status does not show **"Healthy"** after clicking Test | Authentication succeeded but the server rejected the session, or the configured directory is inaccessible | Verify the username has read/write access to the target directory and re-run Test |
| **SFTP (Password Auth)** option is missing from the **File Server** category | Connection type may not be enabled for your workspace | Contact [support@graphium.ai](mailto:support@graphium.ai) to confirm availability |
| **+ Add Connection** button is not available in the vault sidebar | Insufficient vault role | Ask a vault Admin to grant Admin or Builder access |
| Auto-sync toggle cannot be enabled | Connection is in an unhealthy state | Resolve any authentication errors with Test first, then enable Auto-sync |
| Files are not being detected | Incorrect directory path configured | Review the server-specific settings in the connection detail view |
| Sync completes but no records appear in vault tables | Tables are not yet mapped to the SFTP data source | Open the table settings and configure the file source mapping |
| Status flips between Healthy and Error intermittently | Server session timeouts or rate limiting | Check the server's auth log for repeated failed logins and consider allowlisting Graphium AI's IP range |
