## Overview

When you open a record in a vault table, the **record detail view** displays that record's current field values alongside a **completion percentage** — a live indicator of how fully the record has been filled in. After any edit is saved, the detail view reflects the latest values and the recalculated completion percentage.

## When to Use This

Use this feature when you need to:
- Review the current values of a record without opening the edit form
- Check how complete a record is at a glance via the completion percentage
- Open a record's edit form from the detail view to make changes
- Audit which properties on a record have been filled in and which are still blank

## Prerequisites

- You must have access to the vault and table that contains the record
- The vault table must have a schema defined so there are properties to display
- The table must have at least one record to open

## Step-by-Step Instructions

### Step 1: Open the Vault Table

Navigate to the vault that contains the record you want to view. Select the table from the vault sidebar. The **Data** tab opens by default, displaying records in the data grid.

### Step 2: Open a Record

Click a row in the data grid to open the record detail view. The detail surface displays the record's field values as defined by the table's schema and includes the elements listed below.

| Element | Description |
|---------|-------------|
| **Field values** | The current value for each schema property defined on the table |
| **Completion percentage** | A percentage indicator showing how many of the record's fields have been populated |
| **Edit button** | Opens the default form to update the record's values |

> 💡 **Tip:** The detail view is a read-only surface — you cannot edit values directly here. Edits are made through the record's default form, which is opened via the **Edit** button.

### Step 3: Understand the Completion Percentage

The **completion percentage** is a numeric indicator (0–100%) that represents how much of a record has been filled in relative to its schema. It is calculated based on the proportion of user-editable schema properties that have a non-empty value on the record.

For example:
- A record with 10 user-editable schema properties where 5 have values will show **50% complete**
- A record where all user-editable fields are filled in will show **100% complete**
- A brand-new record with no values entered will show **0% complete**

> **Note:** Auto ID and Formula fields are computed by the system and do not count against a record's manual completion. Only user-editable fields factor into the percentage.

### Step 4: (Optional) Edit the Record

To update values shown in the detail view, click the **Edit** button. A slide-in side panel opens with the record's default form pre-populated with current values. See [Editing a Record in the Data Grid](/graphium/data-records/editing-a-record-in-the-data-grid) for details.

After you save changes, the data grid refreshes and the record's completion percentage recalculates based on the new values.

### Step 5: (Optional) Choose Which Properties Appear in the Detail View

Administrators can configure which schema properties are shown in the record detail view and in what order. This is controlled from the table's **Schema** or **Settings** tab — open the table, select the schema/settings surface, and adjust the properties flagged to display in the detail view.

> 💡 **Tip:** Making the detail view show only the properties that matter most to your team reduces scrolling and makes completion percentage easier to reason about at a glance. Check with your vault administrator if you need a property surfaced or hidden.

## What to Expect

| Scenario | What Happens |
|----------|--------------|
| **Record opened** | The detail view displays the record's current field values and completion percentage |
| **Edit saved** | The data grid refreshes, and the completion percentage recalculates to reflect the new values |
| **Edit cancelled** | The form closes with no changes applied; the detail view continues to show the previous values |
| **New record with no values** | The detail view displays empty fields and a 0% completion percentage |

## Common Issues & Troubleshooting

| Issue | Likely Cause | Solution |
|-------|--------------|----------|
| Completion percentage did not update after saving | Not all relevant fields were filled in, or the fields you edited may not be user-editable | Check that the fields you edited have values saved; Auto ID and Formula fields do not affect completion |
| Field values appear stale after saving | The display may not have refreshed | Reload the page to force the detail view to re-fetch the latest record data |
| Completion percentage shows lower than expected | Some schema properties may still be empty | Open the record in edit mode and review which fields remain blank; fill them in and save again |
| A property I expect to see is missing from the detail view | The property may not be configured to display in the detail view | Ask your vault administrator to adjust which properties are surfaced from the table's Schema or Settings tab |
| Completion percentage shows 0% on a record with values | The user-editable fields may all be empty (any visible values could be Auto ID or Formula) | Open the record in edit mode and populate user-editable fields |
