## Overview

When a vault contains more than two tables, the sidebar collapses the list by default — showing the first two tables and a **"X More"** button for the rest. Clicking **X More** reveals all tables at once. Clicking **Show Less** returns the list to its collapsed state.

What you see in the list depends on your vault role: ADMIN and BUILDER users see all tables (including drafts), while CONTRIBUTOR and VIEWER users see only tables that have been enabled for Contributor Mode.

---

## Default View: Collapsed List

When a vault has more than two tables, the TABLES section displays the first two tables in the list and collapses the rest behind an expand link. The expand link shows the count of hidden tables — for example, **"4 More"** if four additional tables are hidden.

Each table row displays the table's name and, when the table has records, a numeric badge showing the total record count (e.g., "55"). Tables with zero records show no badge.

> The tables are ordered by the date they were created, oldest first. The first two tables in that order are shown by default before the list collapses.

---

## How to Expand the Full Tables List

1. In the vault sidebar, locate the **TABLES** section.
2. If the list is collapsed, you will see a **"X More"** button below the visible tables, where X is the number of hidden tables.
3. Click **X More**.
4. All tables in the vault appear in the sidebar list.
5. A **"Show Less"** button appears at the bottom of the list.

---

## How to Collapse the List Again

1. With the full tables list expanded, scroll to the bottom of the TABLES section.
2. Click **Show Less**.
3. The list collapses back to showing the first two tables, and the **"X More"** button returns.

---

## What Tables You See Depends on Your Role

The tables shown in the sidebar are filtered by your vault role:

| Role | Tables Visible |
|------|---------------|
| ADMIN | All tables — both Published and Draft |
| BUILDER | All tables — both Published and Draft |
| CONTRIBUTOR | Only tables with Contributor Mode enabled (Published only) |
| VIEWER | Only tables with Contributor Mode enabled (Published only, read-only) |

If you are a CONTRIBUTOR or VIEWER and do not see a table you expect, it may not have Contributor Mode enabled. Contact your vault ADMIN or BUILDER to enable it.

> Draft tables are visible to ADMIN and BUILDER users only. They appear in the list alongside published tables.

---

## Contributor Mode and the Tables List

When an ADMIN or BUILDER switches the vault to **Contributor Mode** using the toggle at the bottom of the sidebar, the tables list updates to show only the tables that have Contributor Mode enabled — the same view that CONTRIBUTOR and VIEWER users see. Switching back to Builder Mode restores the full table list.

This makes it easy for ADMIN and BUILDER users to preview the vault as a contributor would experience it.

---

## Troubleshooting

| Issue | Likely Cause | What to Do |
|-------|-------------|------------|
| A table is missing from the sidebar | The table does not have Contributor Mode enabled, or your role is CONTRIBUTOR/VIEWER | Ask your vault ADMIN or BUILDER to enable Contributor Mode for that table |
| The "X More" button is not visible | The vault has two or fewer tables visible to your role | No action needed — the list only collapses when there are more than two tables to show |
| The sidebar shows fewer tables than expected after switching to Contributor Mode | Contributor Mode filters to only contributor-enabled tables | Switch back to Builder Mode to see all tables |
| A Draft table is not visible | You have a CONTRIBUTOR or VIEWER role | Draft tables are only visible to ADMIN and BUILDER users |
