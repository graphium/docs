## Overview

The data grid includes a filter panel that lets you narrow down which records are shown based on the values of their properties. You can apply one or multiple filter conditions at once. Filter conditions combine with AND logic — a record must satisfy every condition to appear in the results. Filters can also be saved and reused across sessions.

---

## Opening the Filter Panel

1. Open a vault and select a table from the left sidebar. Ensure you are on the **Data** tab.
2. Click the **Filter** button in the table header bar (between the **Import** button and the **Data / Schema / Settings** tabs).
3. The filter panel opens above the data grid.

Clicking **Filter** again, or closing the panel, hides it. Any active filter conditions remain applied to the grid even when the panel is closed.

---

## The Filter Panel

The filter panel contains the following elements:

| Element | Description |
|---------|-------------|
| **Filters** header | Labels the panel. Displayed with an icon at the top-left. |
| **Saved Filters** button | Opens the saved filters panel to load, save, or manage saved filter sets. |
| **Condition rows** | One row per active filter condition. Each row contains a label, a property selector, an operator selector, a value field, and a remove button. |
| **+ Add condition** button | Adds a new blank condition row to the filter. |

---

## Filter Condition Rows

Each condition row defines a single filter rule:

| Part | Description |
|------|-------------|
| **Where / And** label | The first condition row is labeled **"Where"**. Additional rows are labeled **"And"**, indicating all conditions must be met. |
| **Property dropdown** | Select which property (column) to filter by. Defaults to the first property in the table's schema. |
| **Operator dropdown** | Select the comparison operator (e.g., equals, contains, is empty). Defaults to **"Select operator"**. |
| **Value field** | Enter or select the value to compare against. Appears after an operator is chosen, when applicable. |
| **Remove filter rule** button | Removes this condition row from the filter. |

---

## Adding a Filter Condition

**Steps:**

1. Open the filter panel by clicking the **Filter** button in the table header bar.
2. A default condition row appears. Click the **property dropdown** and select the property you want to filter by.
3. Click the **operator dropdown** and select the comparison operator.
4. Enter or select a value in the **value field** that appears.
5. The grid updates automatically to show only records that match the condition.

---

## Adding Multiple Conditions

To filter by more than one property or rule:

1. Click **+ Add condition** at the bottom of the filter panel.
2. A new condition row is added, labeled **"And"**.
3. Configure the new row's property, operator, and value.
4. The grid updates to show only records that match **all** active conditions.

Repeat to add as many conditions as needed.

---

## Removing a Filter Condition

Click the **Remove filter rule** button (×) at the right end of any condition row. The row is removed and the grid updates immediately. If all condition rows are removed, the filter is cleared and all records are shown.

---

## Saved Filters

The **Saved Filters** button in the filter panel header opens a panel for managing reusable filter sets. Saved filters let you store a named combination of conditions and re-apply them without rebuilding them each time.

For full details on creating, loading, and managing saved filters, see [Using Saved Filters in the Data Grid].

---

## Permissions

All vault member roles — **Admin**, **Builder**, **Contributor**, and **Viewer** — can apply, modify, and remove filter conditions. Filters are per-session and do not affect what other users see.
