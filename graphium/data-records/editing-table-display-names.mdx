## Overview

Each table in Graphium AI has two customizable display names: a **Singular Name** and a **Plural Name**. These names appear throughout the interface to label records in context — for example, the singular name appears on the "Add [Name]" button and in record detail headers, while the plural name appears in table headers and the sidebar.

Display names are set in the **Settings** tab of the table view under the **Table Names** section. Changes save automatically when you move focus out of the field — no separate save button is needed.

> **Note:** The Settings tab is only visible to **ADMIN** and **BUILDER** users. CONTRIBUTOR and VIEWER users cannot see or change this setting.

---

## Before You Begin

- You must have an **ADMIN** or **BUILDER** role in the vault
- Navigate to the table whose display names you want to change
- Confirm you are in the table view with the **Data**, **Schema**, and **Settings** tabs visible in the top-right of the content area

---

## How to Edit the Table Display Names

1. Open the table you want to rename.

2. Click the **Settings** tab in the top-right area of the table view (alongside the Data and Schema tabs).

3. The first section at the top of the Settings tab is **Table Names**. You will see two fields side by side:
   - **Singular Name** — the name used when referring to a single record (e.g., "Item", "Patient", "Inspection")
   - **Plural Name** — the name used when referring to multiple records (e.g., "Items", "Patients", "Inspections")

4. Click into the **Singular Name** field and type the name you want to use for a single record.

5. Click into the **Plural Name** field and type the name you want to use for multiple records.

6. Click or tab out of the field. The name saves automatically — no additional save button is required.

---

## Display Name Rules

| Rule | Detail |
|------|--------|
| Maximum length | 100 characters per field |
| Whitespace | Leading and trailing whitespace is trimmed automatically |
| Both fields required | Singular and plural names must both be set — you cannot save one without the other |
| Default values | New tables default to **Item** (singular) and **Items** (plural) |

---

## Where Display Names Appear

| Name | Where it is used |
|------|-----------------|
| **Singular Name** | The "Add [Name]" button (e.g., "Add Patient"); record detail panel headers |
| **Plural Name** | The table header; the sidebar table label |

---

## What Happens After You Save

- The updated names appear immediately throughout the interface — in the sidebar, the table header, the "Add" button, and record detail panels
- No page reload is required
- The change applies to all users in the workspace

---

## Troubleshooting

| Issue | Likely Cause | What to Do |
|-------|-------------|------------|
| Settings tab is not visible | Your vault role is CONTRIBUTOR or VIEWER | Contact your vault ADMIN — only ADMIN and BUILDER users can access table settings |
| Name does not save after editing | Field may not have lost focus | Click outside the field or press Tab to trigger the save |
| Name reverts after saving | Connectivity or save error | Re-enter the name and ensure a stable network connection before clicking away |
| "Add Item" button still shows old name after change | UI may not have refreshed | Refresh the page — the updated name should appear immediately after |
