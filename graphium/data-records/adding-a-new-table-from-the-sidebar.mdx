## Overview

The **+** button in the TABLES section of the vault sidebar lets ADMIN and BUILDER users create a new table.

> **Note:** The + button is only visible to users with an **ADMIN** or **BUILDER** role. CONTRIBUTOR and VIEWER users do not see this button and cannot create tables.

---

## How to Add a New Table

1. In the vault sidebar, locate the **TABLES** section header.
2. Click the **+** button on the right side of the TABLES header.
3. A **Create Table** modal opens. Enter a **Table Name** (required) — placeholder text shows examples like *"Contacts, Tasks, Inventory"*.
4. Optionally add a **Description** to explain what the table will store.
5. Click **+ Create Table**.

---

## What Happens After the Table Is Created

- The modal closes and you are automatically navigated to the new table's Data view
- The new table appears at the top of the TABLES list in the sidebar, highlighted in teal
- The Data view opens showing *"No records yet"* — the table is empty and ready for records
- The **+ Add [Table Name]** button appears in the top-right, ready for adding records

---

## Renaming the Table

Since the name is set during creation, renaming is done afterwards by clicking the **pencil icon (✏️)** next to the table name in the content header, or via **Settings → Table Names**.

---

## Troubleshooting

| Issue | Likely Cause | What to Do |
|-------|-------------|------------|
| The + button is not visible in the TABLES header | Your vault role is CONTRIBUTOR or VIEWER | Contact your vault ADMIN — only ADMIN and BUILDER users can create tables |
| + Create Table button is greyed out | Table Name field is empty | Enter a table name before clicking + Create Table |
| The table name did not save after typing | The field may not have lost focus | Press Enter or click outside the name field to confirm the rename |
| The new table does not appear in the sidebar | The sidebar may need to refresh | Scroll up in the TABLES list or refresh the page |
