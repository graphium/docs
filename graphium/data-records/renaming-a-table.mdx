## Overview

In Graphium AI, every table within a vault has a name that identifies it in the sidebar, the table header, and throughout the platform. You can rename any table directly from the table view without leaving the page — no modals, no page reloads. The name updates instantly and reflects across the sidebar and all related views as soon as you save.

## When to Use This

Use this article when you need to:

- Rename a table to better reflect its content or purpose after it was created
- Correct a typo or update a name as your vault's structure evolves
- Replace the default "Untitled Table" name assigned when a new table is created

## Prerequisites

- You must be inside a vault that contains at least one table
- You must have a vault role that includes the **Vault Table Update** permission (typically **ADMIN** or **BUILDER**)
- Users with **CONTRIBUTOR** or **VIEWER** roles will not see the edit icon and cannot rename tables

---

## Step-by-Step Instructions

### Step 1: Navigate to the Table

Open the vault containing the table you want to rename. In the left sidebar, click the table name under the **TABLES** section to open its table view.

The table view opens with the **Data** tab active by default, showing the data grid. The table name is displayed as a heading in the top-left of the content area, above the column headers.

### Step 2: Click the Edit Icon

Locate the **pencil icon (✏️)** displayed to the right of the table name in the table header.

Click the pencil icon. The table name immediately transforms from a static heading into an **inline text input field**, pre-filled with the current name and a cursor positioned at the end.

> 💡 **Tip:** The table name input has a blue underline to indicate it is active and ready to edit. No dialog box or modal opens — all editing happens directly in the header.

### Step 3: Type the New Name

Clear the existing name if needed and type the new table name.

> 💡 **Tip:** Table names should be clear and descriptive so team members can identify the table's contents at a glance — for example, "Patient Records", "Inventory Log", or "Inspection Reports".

### Step 4: Save or Cancel

Once you have typed the new name, choose one of the following actions:

- **Press Enter** — Saves the new name immediately. The heading updates in place, and the sidebar also reflects the change.
- **Click anywhere outside the input field** — Also saves the new name, identical to pressing Enter.
- **Press Escape** — Cancels the rename and reverts the table name to what it was before you started editing. No changes are saved.

> ⚠️ **Note:** Clicking outside the input (blur) is treated as a save action, not a cancel. If you want to discard your changes, use **Escape** before clicking away.

---

## What to Expect

After saving:

- The table name updates **instantly in the header** without any page reload.
- The **left sidebar** reflects the new name under the TABLES section immediately.
- The rename is saved to the platform and will persist for all users who access the vault.

If the rename fails (for example, due to a network error), an error notification will appear and the table name will revert to the previous name. No partial updates are applied.

## Common Issues & Troubleshooting

| Issue | Likely Cause | Solution |
|-------|-------------|----------|
| Pencil icon is not visible next to the table name | You have a CONTRIBUTOR or VIEWER role, which does not include Vault Table Update permission | Contact your vault ADMIN to request a BUILDER or ADMIN role |
| Pressing Enter does not save the new name | Possible network connectivity issue | Check your connection and try again. If the issue persists, refresh the page — the name may have saved before the error appeared |
| Table name reverts immediately after saving | The rename API request failed (network error or server issue) | Check your internet connection and retry. If it continues to fail, contact your workspace administrator |
| New name does not appear in the sidebar | UI sync delay | Refresh the page — the name change is saved in the platform and will display correctly after reload |
| I accidentally saved a wrong name by clicking outside the field | Clicking outside the input saves immediately | Click the pencil icon again, type the correct name, and press Enter to correct it |
