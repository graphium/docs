## Overview

New records can be added to any vault table directly from the Data Grid using the table's default form. Clicking **Add {singular}** in the table header opens a slide-in side panel with a dynamic form that automatically generates input fields based on the table's schema. Once submitted, the new record appears in the table immediately.

## When to Use This

Use this feature when you need to:
- Enter a new record into a vault table manually
- Log a new entry in a clinical, operational, or data capture workflow
- Add a record to a table whose data is not managed by an external integration
- Test or populate a table with data during setup or development

> 💡 **Tip:** To bring in many records at once instead of entering them one at a time, use the **Import** button next to **Add {singular}**. Import is covered in its own article.

## Prerequisites

- You must have **Admin**, **Builder**, or **Contributor** access to the vault — Viewer-role users cannot create records and will not see the Add button
- The vault table must have a schema defined (at least one property) so the form has fields to render
- Tables marked **Managed by an integration** (e.g., Graphium EMR) may not support manual record creation — check the table header for this badge
- Your ability to create records is also controlled at the table level. If your administrator has set a table's access to **Off** or **Read-only**, the Add button will not be shown even if your vault role would otherwise allow it

## Step-by-Step Instructions

### Step 1: Open the Vault Table

Navigate to the vault that contains the table where you want to add a record. Select the table from the vault sidebar. The **Data** tab opens by default, displaying the data grid.

### Step 2: Click Add {singular}

In the table header bar, click the **Add {singular}** button. The button label uses the table's configured singular name — for example, **Add Procedure** or **Add Patient**.

> 💡 **Tip:** If you don't see the Add button, your role may be Viewer, the table may be managed by an external integration, or your table-level access is set to Off or Read-only. Check with your vault administrator.

### Step 3: Fill In the Form Fields

A **slide-in side panel** opens from the right side of the screen. The panel contains a form with input fields dynamically generated from the table's schema — every property defined in the schema is available as a field in the form.

By default, the form shows only required fields and any fields your administrator flagged as defaults. Click **Show additional fields (N)** at the bottom of the form to reveal the remaining schema properties.

Fill in the fields for the new record. The form supports all property types used in Graphium vault tables:

| Field Type | How to Fill It In |
|------------|--------------------|
| **Text** | Click the field and type |
| **Number** | Click the field and enter a numeric value |
| **Date** | Uses your browser's native date picker. Appearance and shortcuts vary by browser. |
| **Time** | Uses your browser's native time picker |
| **Date & Time** | Uses your browser's native date-time picker |
| **Boolean** | Toggle the switch or checkbox on or off |
| **Select from List** | Click the field to choose an option from the dropdown |
| **Image** | Click to upload or capture an image |
| **Signature** | Draw your signature in the signature field |
| **Reference** | Click to search records in the referenced table, then select one. The correct record ID and display value are saved together. |
| **Auto ID** | Automatically assigned — no input required |
| **Formula** | Computed from other field values — displayed as read-only |

> 💡 **Tip:** Required fields must be filled in before the form can be submitted. Leave optional fields blank if the information is not yet available — you can always edit the record later.

> **Note:** Auto ID values are generated by the system when the record is saved. If a record is submitted while an Auto ID field still has no value, the system back-fills it before running validation — you will not see a "required" error for Auto ID fields. Formula fields display a computed value based on other fields and cannot be edited directly.

### Step 4: Submit the Record

When you have finished filling in the form, click the **Save** button in the panel footer.

- If all required fields are valid, the record is created and the table list refreshes to show the new entry
- If any required fields are missing or invalid, **inline error messages** appear next to the affected fields — correct them and try submitting again

> 💡 **Tip:** You can dismiss the panel without saving by clicking **Cancel** or clicking outside the panel. No record will be created.

## What to Expect

| Outcome | Details |
|---------|---------|
| **Successful submission** | The side panel closes, the table refreshes, and the new record appears as a row in the data grid |
| **Validation error** | The panel stays open and inline error messages appear next to fields that need attention |
| **Cancel / dismiss** | The panel closes with no record created and no data saved |

## Common Issues & Troubleshooting

| Issue | Likely Cause | Solution |
|-------|--------------|----------|
| The **Add** button is not visible | Your role is Viewer, the table is managed by an external integration, or the table-level access is Off or Read-only | Request Contributor or higher access from your vault administrator, or confirm whether the table supports manual record creation |
| The form panel does not open after clicking **Add** | A temporary loading issue | Wait a moment and try again; refresh the page if the problem persists |
| Required fields are not highlighted after a failed submit | A display or validation sync issue | Scroll through the full form to check for any error messages and correct them before resubmitting |
| The new record does not appear in the grid after saving | The table list refresh may have failed | Reload the page — the record was saved and will appear after a fresh data load |
| A formula field is not calculating correctly | The fields it depends on may be empty | Fill in the source fields that the formula references; the computed value will update automatically |
