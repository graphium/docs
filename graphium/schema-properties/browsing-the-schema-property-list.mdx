## Overview

The **Schema** tab inside any vault table gives you a complete view of every property defined for that table. You can see each property's name, internal identifier, data type, and status at a glance. An inline search bar lets you filter the list by name without leaving the tab or reloading the page.

---

## Opening the Schema Tab

1. Open a vault from the dashboard.
2. Select a table from the left sidebar.
3. Click the **Schema** tab in the top-right of the table view (alongside **Data** and **Settings**).

The schema list loads immediately — no page reload is required.

---

## The Schema Header

The header row above the property list contains several controls:

| Element | Description |
|---------|-------------|
| **Schema [n] fields** | Displays the total count of top-level properties in the table. Child properties nested under a Group are not counted separately. |
| **Read-only** badge | Appears (with a lock icon) when the table's schema is managed by a template and cannot be modified directly. |
| **Search properties...** | An inline search input for filtering the property list by name. Always visible. |
| **+ Add Field** | A teal button for adding a new property. Only shown when the schema is editable (not read-only). |

---

## Reading the Property List

Each row in the property list represents one property. Reading left to right:

| Element | Description |
|---------|-------------|
| Drag handle (⋮⋮) | Appears on hover; drag to reorder the property. |
| Inheritance icon (↩) | A blue chain-link indicator meaning this property is inherited from a vault template. |
| Expand/collapse (∨) | Appears on Group-type properties. Click to show or hide the group's child properties. |
| Type icon | A small icon indicating the property's data type (see [Type Icons](#type-icons) below). |
| **Display name** | The human-readable label used in forms and data views. |
| `internal_name` | The system identifier shown in gray. Set at creation and cannot be changed. |
| **Required** badge | A red badge shown when the property is marked as required. |
| Type label | The property's data type displayed as text on the right (e.g., Text, Number, Date). |
| **Edit** | Opens the property for editing. Only visible on editable tables. |
| Trash icon | Removes the property (with confirmation). Only visible on editable tables. |

---

## Type Icons

Each property displays a small icon that identifies its data type at a glance:

| Icon | Type |
|------|------|
| **T** | Text |
| **#** | Number |
| Calendar | Date |
| Clock | Time |
| Box/folder | Group (Object) |
| Chain link | Reference |
| Image frame | Image |
| Pen/signature | Signature |
| List lines (≡) + base icon | Collection — appears alongside the base type icon when "Allow multiple values" is enabled (e.g., Text[] shows both ≡ and T) |

## Grouped (Object) Properties and Hierarchy

When a table contains a **Group**-type property, its child properties are displayed indented beneath it. This lets you see the nesting structure of complex schemas at a glance.

- The Group row shows an expand/collapse chevron (∨). Clicking it collapses or expands all child properties under that group.
- Child properties are visually indented to show they belong to the parent group.
- The total field count in the header reflects only top-level properties — child properties are not counted separately.

**Example:** A "Patient Demographics" Group property with three children (Date of Birth, Gender, Insurance) counts as 1 field in the header, with all three children shown indented beneath it.

---

## Archived Properties

By default, archived properties are hidden from the schema list. To view them:

- Enable the **Show Archived** toggle in the schema header.

Archived properties appear with a greyed-out background and remain in their original position in the list. Their data is preserved even though they no longer appear in forms or the data grid.

---

## Searching Properties by Name

The **Search properties...** bar at the top of the schema list lets you filter properties by display name in real time. Filtering is client-side — no network request is made and the page does not reload.

**How to search:**

1. Click the **Search properties...** input in the schema header.
2. Start typing any part of a property's display name. The list updates instantly as you type.
3. To clear the filter, delete the text in the search bar or press **Escape**.

**Search behavior:**

- Matching is **case-insensitive** — typing "date" matches "Case Date", "Date of Birth", and "DATE".
- **When a top-level property matches:** It appears in the results. If it is a Group type, all its child properties are shown beneath it with their original indentation, so the hierarchy is preserved.
- **When only a child property matches:** The child is shown in the results with its parent Group displayed above it for context. This ensures you always know which group the property belongs to.
- **No match:** The list displays the message "No fields match your search."

**Example — searching "patient":**

The filter returns "Patient Name" (a direct top-level match) and "Patient Demographics" (another top-level match) along with all three of its children — Date of Birth, Gender, and Insurance — indented beneath it, preserving the group hierarchy.

---

## Permissions

Viewing the schema list requires the **vault_table_view** permission. All standard roles (Builder, Contributor, and above) can view the schema. The **+ Add Field** and **Edit** controls are only visible to users with **vault_table_update** permission.
