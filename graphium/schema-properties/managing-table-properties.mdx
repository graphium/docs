## Overview

Every table in Graphium AI has a **schema** — the set of properties (fields) that define what data each record can hold. From the Schema tab, you can view all existing properties, add new ones, edit them, and search through them.

---

## Opening the Schema Tab

1. Open a vault and select a table from the sidebar.
2. Click the **Schema** tab in the top-right of the table view (alongside **Data** and **Settings**).

The schema list loads immediately with no page reload required.

> **Note:** If a table's schema is marked **Read-only**, you will see a "Read-only" badge and will not be able to add or edit properties directly.

---

## The Schema Header

| Element | Description |
|---------|-------------|
| **Schema [n] fields** | Total count of top-level properties. Child properties in groups are not counted separately. |
| **Read-only** badge | Appears when the schema is managed by a template and cannot be modified. |
| **Search fields...** | Inline search input for filtering the property list by name in real time. |
| **+ Add Field** | Button for adding a new property. Only visible on editable schemas. |
| **Show Archived** | Toggle to reveal archived properties. |

---

## Reading the Property List

Each row in the property list represents one property:

| Element | Description |
|---------|-------------|
| Drag handle (⋮⋮) | Drag to reorder properties |
| Type icon | Visual indicator of the property's data type |
| **Display name** | The human-readable label shown in forms and data views |
| `internal_name` | The system identifier shown in gray — set once and cannot be changed |
| **Required** badge | A red badge shown when the property is marked as required |
| Type label | The property's data type (e.g., Text, Number, Date) |
| **Edit** | Opens the property for editing |
| Trash icon | Archives or deletes the property |

---

## Adding a New Property

1. Click the **+ Add Field** button in the top-right corner of the Schema tab.
2. In the **Add Property** modal, fill in:
   - **Name** — The display label shown throughout the platform
   - **Description** *(optional)* — Up to 128 characters
   - **Internal Name** — Auto-generated from the display name; can be edited before saving but cannot be changed after creation
   - **Property type** — Select from the dropdown
   - **Allow multiple values** *(optional)* — Stores an array of values instead of a single value
   - **Required** *(optional)* — Makes the property mandatory
3. Click **Add** to save.

> **Note:** Once a property has been created with data, only the display name and description can be edited. The internal name, property type, and collection setting are locked after creation.

---

## Editing an Existing Property

1. Click the **Edit** button on the property's row.
2. Update the **display name**, **description**, or **Required** setting.
3. Click **Save**.

The internal name, property type, and "Allow multiple values" setting cannot be changed after creation. If you need to change these, archive the existing property and create a new one.

---

## Searching Properties

Click the **Search fields...** bar and start typing. The list filters in real time — matching is case-insensitive and group hierarchy is preserved in results.

---

## Archived Properties

Archived properties no longer appear in forms or the data grid, but their data is preserved. Enable the **Show Archived** toggle to view them. Archived properties appear with a greyed-out background and can be unarchived or permanently deleted from this view.

---

## Permissions

| Action | Required Role |
|--------|--------------|
| View schema | Any vault member |
| Add or edit properties | Vault Admin or Builder |
