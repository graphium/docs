## Overview

This article is a reference guide for each simple property type in Graphium. For instructions on opening the schema editor and using the Add Property form, see [Adding Simple Table Property Types](/graphium/schema-properties/adding-simple-table-property-types).

---

## Text

**What it is:** A plain text string. Use it for names, freeform identifiers, notes, and any value that doesn't fit a more specific type.

**Examples:** Patient Name, MRN, Gender, Insurance carrier

**How to configure:**
1. In the Add Property modal, enter a **Name**.
2. From the **Property type** dropdown, select **Text**.
3. Optionally enable **Allow multiple values** if the field will hold more than one text entry.
4. Click **Add**.

No additional configuration is required. Once saved, Text properties appear with a **T** icon in the schema list.

---

## Number

**What it is:** A numeric value — either integer or decimal. Use it for quantities, durations, measurements, and any field where arithmetic or comparison operations will be needed.

**Examples:** Procedure Duration (minutes), Base Units, Anesthesia Time, Dosage Amount

**How to configure:**
1. In the Add Property modal, enter a **Name**.
2. From the **Property type** dropdown, select **Number**.
3. Optionally enable **Allow multiple values** if the field will hold more than one number.
4. Click **Add**.

No additional configuration is required. Number properties appear with a **#** icon in the schema list.

---

## Date

**What it is:** A calendar date (no time component). Use it for anything tied to a specific day.

**Examples:** Case Date, Date of Birth, Discharge Date

**How to configure:**
1. In the Add Property modal, enter a **Name**.
2. From the **Property type** dropdown, select **Date**.
3. Optionally enable **Allow multiple values**.
4. Click **Add**.

No additional configuration is required. Date properties appear with a **calendar** icon in the schema list.

---

## Time

**What it is:** A time-of-day value, separate from any date. Use it when you need to record a clock time without associating it with a specific calendar date.

**Examples:** Start Time, End Time, Scheduled Time

**How to configure:**
1. In the Add Property modal, enter a **Name**.
2. From the **Property type** dropdown, select **Time**.
3. Optionally enable **Allow multiple values**.
4. Click **Add**.

No additional configuration is required. Time properties appear with a **clock** icon in the schema list.

---

## Date/Time

**What it is:** A combined date and time value. Use it when both the calendar date and the time of day are relevant and should be stored together.

**Examples:** Created At, Submitted At, Event Start

**How to configure:**
1. In the Add Property modal, enter a **Name**.
2. From the **Property type** dropdown, select **Date/Time**.
3. Optionally enable **Allow multiple values**.
4. Click **Add**.

No additional configuration is required. Date/Time properties appear with a **calendar** icon in the schema list.

---

## Boolean

**What it is:** A true/false flag. Use it for yes/no conditions, binary status indicators, and any field where the only meaningful values are two opposing states.

**Examples:** Verified, Active, Consented, Requires Follow-up

**How to configure:**
1. In the Add Property modal, enter a **Name**.
2. From the **Property type** dropdown, select **Boolean**.
3. Optionally enable **Allow multiple values** (uncommon for Boolean fields).
4. Click **Add**.

No additional configuration is required.

---

## Image

**What it is:** A field for storing photographs, scanned documents, or any visual record attached to a data entry.

**Examples:** Charge Ticket, Consent Form Scan, Wound Photo

**How to configure:**
1. In the Add Property modal, enter a **Name**.
2. From the **Property type** dropdown, select **Image**.
3. Optionally enable **Allow multiple values** if the record may have more than one image.
4. Click **Add**.

No additional configuration is required.

---

## Signature

**What it is:** A digital signature capture field. Use it for compliance and authorization workflows where a provider or patient needs to sign off on a record.

**Examples:** Provider Signature, Patient Authorization, Attending Physician Sign-off

**How to configure:**
1. In the Add Property modal, enter a **Name**.
2. From the **Property type** dropdown, select **Signature**.
3. Click **Add**.

No additional configuration is required. Signature properties do not support multiple values.

---

## Select from List

**What it is:** A property that restricts input to a predefined set of options. Use it when you want to enforce consistent, controlled vocabulary across records.

**Examples:** Contact Type, Status, Procedure Category, Insurance Plan

**How to configure:**
1. In the Add Property modal, enter a **Name**.
2. From the **Property type** dropdown, select **Select from List**.
3. In the **List Options** section, type each option and press **Enter** or click **+ Add**. At least one option is required.
4. To remove an option, click the trash icon next to it.
5. Optionally enable **Allow multiple values** to allow selecting more than one option per record.
6. Click **Add**.

> **Note:** List options can be added or removed after the property is created by clicking **Edit** on the property row.

---

## Object

**What it is:** A named container that holds child properties, creating a hierarchical nesting structure within the schema. An Object property does not store data itself — it organizes related fields together.

**Examples:** Patient Demographics (containing Date of Birth, Gender, Insurance), Address (containing Street, City, State, Zip)

**How to configure:**
1. In the Add Property modal, enter a **Name** for the group (e.g., *Patient Demographics*).
2. From the **Property type** dropdown, select **Object**.
3. Click **Add**. The Object property appears in the schema list with a **folder** icon.
4. To add child properties, click the expand arrow next to the Object row, then click **+ Add Field** for each child property.

> Object properties do not support the "Allow multiple values" option.

---

## Editing a Property

To edit an existing property, click the **Edit** button on its row in the schema list. You can update the name and description freely. The property type, internal name, and "allow multiple values" setting are locked after creation.
