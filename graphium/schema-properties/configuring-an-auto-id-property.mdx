## Overview

An Auto ID property automatically generates a unique, sequential identifier for every record added to a vault table. When a new record is created, the system issues the next value in the sequence and formats it using the configured prefix, suffix, and number length — no manual entry required.

Auto ID is useful for record identifiers that need to be human-readable, consistently formatted, and guaranteed unique — for example, case numbers (`CASE-000001`), invoice numbers (`INV-00001-2026`), or provider IDs (`PRV-0001`).

Auto ID properties cannot store multiple values and are always system-managed. Once the starting number has been set and the property published, it cannot be changed. Only users with a **Builder** or **Admin** vault role can add or edit Auto ID properties.

## Configuring an Auto ID Property

### Step 1 — Open the Schema Editor

1. In the left sidebar, select the vault table you want to configure.
2. Click the **Schema** tab at the top of the table view.
3. Click **+ Add Field** in the top-right corner of the schema list.

The **Add Property** modal opens.

### Step 2 — Fill in the standard fields

| Field | Required | Notes |
|---|---|---|
| **Name** | Yes | The display label shown in the data grid and forms (e.g., `Invoice Number`) |
| **Description** | No | Optional context about the property (0 / 128 characters) |
| **Internal Name** | Auto-generated | Snake_case version of the name (e.g., `invoice_number`). Locked after the property is saved and cannot be changed |

### Step 3 — Select the Auto ID type

In the **Property type** dropdown, search for or select **Auto ID**.

> Auto ID appears with a **New** badge in the property type list.

Once selected, an **Auto ID Configuration** panel appears directly below the Property type field.

<Frame>
  <img src="/images/graphium/tables/autoid-property-type.png" alt="Auto ID property type" />
</Frame>

### Step 4 — Configure the Auto ID settings

The Auto ID Configuration panel contains four fields:

| Field | Required | Default | Description |
|---|---|---|---|
| **Prefix** | No | *(empty)* | Text prepended to every generated value (e.g., `INV-`) |
| **Suffix** | No | *(empty)* | Text appended to every generated value (e.g., `-2026`) |
| **Starting Number** | Yes | `1` | The first numeric value issued when the first record is created. **Cannot be changed after the property is saved.** |
| **Number Length (digits)** | Yes | `6` | The total number of digits in the numeric portion. Shorter values are zero-padded to this length |

<Frame>
  <img src="/images/graphium/tables/autoid-configuration-settings.png" alt="Auto ID configuration settings" />
</Frame>

#### Live Preview

As you configure the fields, a **Preview** panel updates in real time:

- **Current** — The value that will be issued to the first record (e.g., INV-00001-2026)
- **Next** — The value that will be issued to the second record (e.g., INV-00002-2026)

#### Formatting Example

With the following configuration:

| Setting | Value |
|---|---|
| Prefix | `INV-` |
| Suffix | `-2026` |
| Starting Number | `1` |
| Number Length | `5` |

The first record receives INV-00001-2026, the second receives INV-00002-2026, and so on.

> **Note:** The Allow Multiple Values option is not available for Auto ID properties. Each property always holds a single, system-generated value.

### Step 5 — Add the property

Click **Add** to save the property. The system registers the Auto ID property in the table schema and creates an internal sequence tracker to manage numeric incrementation.

---

## What Happens After Saving

- The Auto ID property appears as a new column in the data grid.
- When a new record is created in the table, the system automatically generates the next formatted value and populates the field — no user input is needed or accepted.
- Values are guaranteed to be unique: the sequence is incremented atomically, and a failed record creation rolls back the sequence so no numbers are skipped or duplicated.

## Editing an Auto ID Property

After a property is saved, you can edit it inline from the Schema tab by clicking **Edit** next to the property row. The following rules apply:

| Field | Editable after publish? |
|---|---|
| Name | Yes |
| Description | Yes |
| Internal Name | No — locked permanently after first save |
| Prefix | Yes |
| Suffix | Yes |
| Starting Number | **No** — locked permanently once the sequence is created |
| Number Length (digits) | Yes |

> **Important:** Changing the Prefix, Suffix, or Number Length after records have already been created will affect the format of future records only. Existing records retain the value they were issued at creation.

---

## Archiving an Auto ID Property

To remove an Auto ID property from active use, archive it from the Schema tab. Archiving is reversible — the property definition and existing record values are preserved. See [Adding Simple Table Properties](/graphium/schema-properties/adding-simple-table-property-types) for archiving and unarchiving instructions.

## Troubleshooting

| Issue | Likely Cause | What to Do |
|-------|-------------|------------|
| Auto ID value is not generated on a new record | The property may not have been saved correctly | Check the schema to confirm the Auto ID property exists; if missing, re-add it |
| Starting Number cannot be changed | It is locked after the property is first published | Archive the property and create a new one with the correct starting number |
| Values have gaps in the sequence | A record creation may have failed and rolled back | This is expected behaviour — no numbers are duplicated |
| Prefix or suffix changes aren't reflected on existing records | Changes only affect future records | Existing records retain the value issued at creation |
