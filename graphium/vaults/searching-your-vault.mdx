## Overview

The vault search bar lets you find any data inside your vault without manually navigating through tabs or scrolling through tables. It is available from the top navigation header whenever you are inside a vault, and opens a full command palette overlay where you can search across records, tables, schema fields, connections, and assistants in one place.

---

## When to Use This

Use vault search when you need to:

- Quickly locate a specific patient record, case, or entry by name, ID, or any field value
- Find a table by name without scrolling through the sidebar
- Look up a schema field definition to check its type, description, or whether it is required
- Navigate directly to a connection without going through the Automations tab

---

## Prerequisites

- You must be an active member of the vault you want to search. All vault roles can use search: **ADMIN**, **BUILDER**, **CONTRIBUTOR**, and **VIEWER**.
- You must be inside a vault — the search bar is not available on the workspace or vault list pages.

---

## Step-by-Step Instructions

### Step 1: Open the Search Overlay

While inside a vault, open the search overlay using either method:

- **Click the search bar** in the top navigation header. It displays your vault name as a placeholder — for example, *Search Cases - Community Surgery Center...*.
- **Press `⌘K`** on Mac or **`Ctrl+K`** on Windows/Linux from anywhere inside the vault.

The search overlay opens as a centered modal with a blurred backdrop. The search input is automatically focused.

> **Note:** The search bar only appears when you are inside a vault. If you do not see it, navigate into a vault first.

---

### Step 2: Type Your Search Query

Begin typing in the search input. Results appear automatically as you type — no need to press Enter.

> **Note:** A minimum of **3 characters** is required before results are shown. This keeps results relevant and reduces noise.

Results are grouped by category: **Records**, **Schema**, **Tables**, **Connections**, and **Assistants**. Each result row shows:

- An **icon** identifying the category
- A **title** — the primary name or matched value, with your search term shown in **bold**
- A **subtitle** — secondary context such as the source table, field type, or connection type
- An optional **badge** — for example, *Required* on schema fields or *Healthy / Error* on connections

---

### Step 3: Filter by Category (Optional)

A row of **category filter pills** appears above the results. Each pill shows the number of matches found in that category.

- By default, the **All** pill is active and shows results across all categories.
- Click any pill — **Records**, **Schema**, **Tables**, **Connections**, or **Assistants** — to filter results to that category only.
- Click **All** again to return to the multi-category overview.

> 💡 **Tip:** If you know what type of item you are looking for, select a category pill first. You will see far more results and can refine your query from there.

---

### Step 4: Navigate to a Result

**Using the mouse:** Click any result row to navigate directly to that item.

**Using the keyboard:**

| Key | Action |
|-----|--------|
| `↑` / `↓` | Move up or down through results |
| `Enter` | Open the selected result |
| `Esc` | Close the search overlay |

The selected row is highlighted with a cyan left border and scrolls into view automatically.

---

### Step 5: Close the Search Overlay

When you are done, close the overlay by pressing `Esc` or clicking anywhere outside it.

---

## What to Expect

Where you land depends on the type of result you selected:

| Result Type | What Happens |
|-------------|--------------|
| **Record** | The record opens directly in a detail panel where you can view and edit its fields. |
| **Schema field** | The Tables tab opens in schema view for the table that contains the field. |
| **Table** | The Tables tab opens with that table selected and its data view loaded. |
| **Connection** | The connection detail view opens directly. |
| **Assistant** | The AI Assistant page opens. |

> **Note:** Record search matches against **field values only** — not column names. For example, searching *Patient Name* will return schema results (the field definition), but not every record in the table. To find a specific record, search for the actual data value, such as a patient's name or MRN number.

---

## Recent Searches

When you open the search overlay without typing anything, your **recent search history** for the current vault is displayed.

- Recent searches are **per vault** — your history in one vault does not appear in another.
- Up to **15 recent searches** are saved per vault. Older entries are dropped automatically as new ones are added.
- Recent searches are stored in **your browser** — they are not synced across devices.
- Click any recent search term to re-run it instantly.
- Click **Clear all** to remove all recent searches for the current vault, or click **✕** next to any entry to remove it individually.

---

## Permissions

Search is available to all vault roles and is read-only — it does not create, edit, or delete any data.

| Vault Role | Can Use Search |
|------------|----------------|
| ADMIN | ✅ |
| BUILDER | ✅ |
| CONTRIBUTOR | ✅ |
| VIEWER | ✅ |

Search results are always scoped to the vault you are currently viewing. Records, tables, schema, and assistants return results from that vault only. Connections are scoped to your workspace and are visible from any vault within it.

## Common Issues & Troubleshooting

| Issue | Likely Cause | What to Do |
|-------|--------------|------------|
| The search bar is not visible | You are not inside a vault | Navigate into a vault — the search bar only appears in the vault header |
| No results appear after typing | Query is fewer than 3 characters | Type at least 3 characters before results will show |
| Searching a column name returns no records | Record search matches values only, not column names | Search for the actual data value — e.g., a patient name or MRN — instead of the column label |
| A record result does not open | The record may have been deleted or the vault data may have changed | Try searching again; if the problem persists, contact your vault admin |
| Category pills show 0 for every category | The vault may have very little data, or your query is too specific | Try a shorter or more general query; confirm you are searching the correct vault |
