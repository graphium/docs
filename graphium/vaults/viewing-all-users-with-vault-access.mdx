## Overview

The **Access & Permissions** page in vault settings controls who can access the vault and what they can do. It contains two tabs — **Users** and **Teams** — and is accessible to Vault Admins and Builders from the vault's Settings sidebar.

The **Users** tab lists every person who has been granted access to the vault individually, along with their role and invitation status.

---

## How to Open the Users List

1. Open the vault.
2. In the left sidebar, click **Settings**.
3. Click **Access & Permissions** in the settings sub-navigation.
4. The **Users** tab is selected by default.

---

## What You See on the Access & Permissions Page

### Tabs

| Tab | Description |
|-----|-------------|
| **Users** | Lists individual users with vault access |
| **Teams** | Lists teams with vault access |

### Add User

At the top of the Users tab, an **Add User** button allows you to invite a new user by email or add an existing workspace member. Helper text below the button reads: *"Enter an email to invite a new user or add an existing workspace member."*

### User List

Each row in the list represents one person with access to the vault.

| Element | Description |
|---------|-------------|
| **Avatar** | A colored circle containing the user's initials |
| **Name** | The user's full name |
| **Status badge** | A **"Pending"** label appears next to the name if the user has not yet accepted their invitation |
| **Email** | The user's email address |
| **Role** | A dropdown showing the user's current vault role — see [Understanding Vault User Roles](Understanding%20Vault%20User%20Roles.md) |
| **Remove** | A trash icon to remove the user from the vault |

Users without a status badge have **Active** access. Users with a **"Pending"** badge have been invited but have not yet accepted.

### Role Permissions

Below the user list, a **Role Permissions** legend describes what each vault role can do:

| Role | Description |
|------|-------------|
| **Admin** | Full access to manage vault, users, and all data |
| **Builder** | Can create and modify tables, forms, and workflows |
| **Contributor** | Can add and edit data, but cannot modify structure |
| **Viewer** | Read-only access to view data |

---

## Permissions

| Role | Can View the Users List |
|------|------------------------|
| Admin | ✅ |
| Builder | ✅ |
| Contributor | ❌ |
| Viewer | ❌ |
